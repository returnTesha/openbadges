# OpenBadge 프로젝트 개요

## Open Badges란?

**1EdTech**(구 IMS Global)가 만든 **디지털 배지 국제 표준**이다.
"누가, 누구에게, 무엇을, 언제 인정했는지"를 **암호학적으로 증명**할 수 있는 디지털 인증서.

쉽게 말하면:
- 종이 자격증 → 위조 가능, 발급기관 없어지면 확인 불가
- Open Badge → **발급기관이 망해도 수학적으로 진짜인지 검증 가능**

## OB 2.0 vs OB 3.0

| 구분 | OB 2.0 | OB 3.0 |
|------|--------|--------|
| 기반 | 자체 스펙 | **W3C Verifiable Credentials** |
| 검증 방식 | 발급자 서버에 JSON 호스팅 or JWS 서명 | **DataIntegrityProof (Ed25519)** or JWT |
| 발급자 죽으면? | 호스팅 방식이면 검증 불가 | **서명이 남아있으므로 여전히 검증 가능** |
| 수령자 식별 | 이메일 평문 | 해시된 이메일 or **DID** |
| 확장성 | 제한적 | W3C VC 생태계 전체와 호환 |

## 핵심 구성 요소 (3개)

```
┌──────────────────────────────────────────────────────┐
│                  AchievementCredential                │
│              (= Verifiable Credential)                │
│                                                      │
│  ┌─────────────┐  ┌──────────────┐  ┌──────────────┐│
│  │   Issuer     │  │  Achievement │  │  Recipient   ││
│  │  (발급자)    │  │  (배지 정의)  │  │  (수령자)    ││
│  │             │  │              │  │              ││
│  │ - name      │  │ - name       │  │ - id (DID)   ││
│  │ - url       │  │ - description│  │ - email hash ││
│  │ - publicKey │  │ - criteria   │  │              ││
│  └─────────────┘  │ - image      │  └──────────────┘│
│                   └──────────────┘                   │
│                                                      │
│  ┌──────────────────────────────────────────────────┐│
│  │                    Proof                          ││
│  │  (암호학적 증명 — 이게 핵심)                      ││
│  │                                                  ││
│  │  - type: DataIntegrityProof                      ││
│  │  - cryptosuite: eddsa-rdfc-2022                  ││
│  │  - proofValue: z3fFfxgwjy1... (Ed25519 서명)     ││
│  └──────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────┘
```

1. **Issuer (발급자)**: 배지를 발급하는 기관. 공개키를 가지고 있음
2. **Achievement (배지 정의)**: 어떤 역량/성취를 인정하는지 정의
3. **Proof (증명)**: 발급자의 비밀키로 만든 **디지털 서명**

## 이 프로젝트가 하는 일

```
[관리자] → 발급자 등록, 배지 정의
                ↓
[발급자] → 수령자에게 배지 발급 (서명 생성)
                ↓
[수령자] → 배지를 받아서 보관 (JSON 파일 or 지갑)
                ↓
[검증자] → 배지가 진짜인지 검증 (서명 검증)
```

- **발급**: 배지 데이터를 만들고, Ed25519 비밀키로 서명
- **검증**: 서명된 배지를 받아서, 발급자의 공개키로 서명이 맞는지 확인
- **취소**: 발급자가 취소 목록에 등록하면 더 이상 유효하지 않음

## 기술 스택

| 구성 | 기술 |
|------|------|
| 언어 | Go |
| 웹 프레임워크 | Fiber v2 |
| DB | PostgreSQL |
| 파일 저장소 | MinIO (배지 이미지) |
| 서명 알고리즘 | **Ed25519 (eddsa-rdfc-2022)** |
| 데이터 포맷 | JSON-LD (W3C Verifiable Credentials) |
| 설정 | TOML + 환경변수 |
