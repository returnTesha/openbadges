## 슬라이드 2: 현재 문제점

**종이 수료증의 한계**

- 위조·변조가 쉽다
- 발급 기관에 매번 진위 확인 요청해야 한다
- 분실하면 재발급 절차가 번거롭다
- 온라인 제출이 어렵다 (스캔해서 PDF로?)
- 발급 기관이 사라지면 증명이 불가능하다

---

## 슬라이드 3: 해결책 — Open Badges 3.0

**국제 표준 디지털 배지**

- W3C Verifiable Credentials 기반 국제 표준
- 1EdTech (구 IMS Global)에서 제정
- 전 세계 교육기관, 기업에서 채택 중
- JSON 데이터에 수학적 서명을 넣어서 위변조 불가능

핵심: "이 배지를 정말 이 기관이 발급했다"를 수학으로 증명

---

## 슬라이드 4: OB 2.0 vs OB 3.0

| 구분 | OB 2.0 (구버전) | OB 3.0 (우리가 구현) |
|------|-----------------|---------------------|
| 표준 | 자체 스펙 | W3C Verifiable Credentials |
| 검증 방식 | 발급서버에 접속해서 확인 | 수학적 서명 검증 (서버 불필요) |
| 발급기관 서버 다운 시 | 검증 불가능 | 여전히 검증 가능 |
| 수령자 식별 | 이메일 평문 | 이메일 해시 (프라이버시 보호) |
| 확장성 | 제한적 | W3C 생태계 전체와 호환 |

## 슬라이드 5: 핵심 기술 — Ed25519 디지털 서명

**발급 (서명 생성)**

```
배지 JSON 문서
    ↓
정규화 (RDFC-1.0) — 같은 데이터 → 항상 같은 바이트
    ↓
SHA-256 해시 — 문서 → 32 bytes 지문
    ↓
Ed25519 Sign(비밀키) — 수학적 서명 생성
    ↓
proof 블록에 서명값 저장
```

**검증 (서명 확인)**

```
서명된 배지 수신
    ↓
같은 방식으로 정규화 + 해시
    ↓
발급자 공개키 조회
    ↓
Ed25519 Verify(공개키, 해시, 서명) → true/false
```

발급자 서버 없이, 공개키만 있으면 누구나 검증 가능

## 슬라이드 6: 이미지 유실 문제와 해결

**문제: 기존 URL 방식**

```json
"image": { "id": "https://example.com/badge.png" }
```
- 서버 다운 → 이미지 404 유실
- 서명은 URL 문자열만 보호, 실제 이미지는 보호 못 함

**해결: Base64 Data URI 임베딩**

```json
"image": { "id": "data:image/svg+xml;base64,PHN2Zy..." }
```
- 이미지 데이터가 JSON 안에 직접 포함
- 서명이 이미지 데이터 자체를 보호
- 서버 없어도 이미지 표시 가능
- 이미지 위변조 시 서명 검증 실패

## 슬라이드 7: 적용 시나리오 — 비교과 수료 배지

**"AI·SW 비교과 프로그램 수료" 배지 발급 흐름**

1. 학생이 비교과 활동 4회 이상 참여 + 결과물 제출
2. 교육원에서 수료 확인
3. 시스템이 자동으로 디지털 배지 생성
4. Ed25519 서명 → 위변조 불가능한 배지 완성
5. 학생에게 배지 전달 (JSON 파일 또는 링크)
6. 학생이 취업 포트폴리오, LinkedIn 등에 제출
7. 기업이 공개키로 즉시 검증 — 기관에 연락할 필요 없음

## 슬라이드 9: 실제 발급 결과 — 서명 후 (Signed Credential)

서명 전 문서에 proof 블록이 추가되어 최종 배지가 완성됩니다.

```json
{
  "...위 내용 전체...",

  "proof": {
    "type": "DataIntegrityProof",
    "created": "2026-03-11T23:16:28Z",
    "verificationMethod": "did:web:badge.kopo.ac.kr:issuers:gwangmyeong#key-1",
    "cryptosuite": "eddsa-rdfc-2022",
    "proofPurpose": "assertionMethod",
    "proofValue": "z3DVx5UWbke4VyNvy65tZ2pajzfMKwEy8LjxW2GCHHqXGdjnSVCq3YnRumfdLaMJDfgokxLVuU3muxHSjNy554rBg"
  }
}
```

- proofValue: Ed25519 서명값 (base58-btc 인코딩, 64 bytes)
- verificationMethod: 검증에 사용할 공개키 위치 (DID URL)
- cryptosuite: eddsa-rdfc-2022 (Ed25519 + RDFC-1.0 정규화)

## 슬라이드 10: 실제 검증 테스트 결과

Go 테스트 코드 실행 결과 (PASS):

| 테스트 | 시나리오 | 결과 |
|--------|----------|------|
| 검증 1 | 원본 배지 그대로 검증 | true — 정상 배지 확인 |
| 검증 2 | 배지 이름을 "석사 학위"로 위변조 | false — 위변조 감지 |
| 검증 3 | 다른 사람의 키로 가짜 서명 | false — 가짜 서명 감지 |

핵심 포인트:
- 내용 1비트라도 바뀌면 서명 검증 실패
- 발급자의 비밀키 없이는 유효한 서명 생성 불가능
- 발급 기관 서버 없이도 공개키만으로 검증 가능

## 슬라이드 12: 시스템 아키텍처

```
[학생]  ←── 배지 수신 ──  [배지 발급 API 서버 (Go/Fiber)]
                              │
                    ┌─────────┼──────────┐
                    │         │          │
               [PostgreSQL] [MinIO]  [Ed25519 서명엔진]
               배지/발급자   이미지    키관리·서명·검증
               메타데이터    저장소
                              │
[기업/검증자]  ──  검증요청  ──┘
     │
     └── 공개키로 직접 검증 (서버 불필요)
```

기술 스택: Go 1.25 / Fiber v2 / PostgreSQL / MinIO / Ed25519

## 슬라이드 13: 기대 효과

**교육기관**
- 수료증 발급·관리 비용 절감
- 위변조 민원 제로
- 국제 표준 준수 (1EdTech OB 3.0)

**학생**
- 디지털 포트폴리오에 즉시 활용
- 취업 시 신뢰도 높은 역량 증명
- 평생 보관 가능 (종이 분실 걱정 없음)

**기업/검증자**
- 발급 기관에 연락 없이 즉시 검증
- 자동화된 자격 확인 가능
- 국제 표준이라 해외 인증도 호환

## 슬라이드 14: 마무리

**요약**

- Open Badges 3.0 = W3C 국제 표준 디지털 배지
