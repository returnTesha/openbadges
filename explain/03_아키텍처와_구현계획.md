## API 엔드포인트

### Issuer (발급자) 관리
```
POST   /api/v1/issuers          발급자 생성 (키 쌍 자동 생성)
GET    /api/v1/issuers           발급자 목록
GET    /api/v1/issuers/:id       발급자 상세 (공개키 포함)
PUT    /api/v1/issuers/:id       발급자 수정
DELETE /api/v1/issuers/:id       발급자 삭제
```

### BadgeClass (배지 정의) 관리
```
POST   /api/v1/badges           배지 정의 생성
GET    /api/v1/badges            배지 목록
GET    /api/v1/badges/:id        배지 상세
PUT    /api/v1/badges/:id        배지 수정
DELETE /api/v1/badges/:id        배지 삭제
```

### Assertion (발급/검증) — ★ 핵심
```
POST   /api/v1/assertions                배지 발급 (서명 생성)
GET    /api/v1/assertions/:id             발급 내역 조회
GET    /api/v1/assertions/:id/verify      배지 검증 (서명 검증)
GET    /api/v1/assertions/:id/credential  OB 3.0 JSON-LD 출력
POST   /api/v1/assertions/:id/revoke      배지 취소
```

## 현재 구현 상태

| 항목 | 상태 | 설명 |
|------|------|------|
| 프로젝트 골자 | ✅ 완료 | EOH 패턴 기반 디렉토리 + 빌드 통과 |

| Ed25519 키 생성 | ❌ 미구현 | Issuer 생성 시 키 쌍 자동 생성 |
| JSON-LD 정규화 (RDFC-1.0) | ❌ 미구현 | 서명 전 필수 단계 |
| Ed25519 서명 생성 | ❌ 미구현 | proofValue 생성 |
| Ed25519 서명 검증 | ❌ 미구현 | verify 엔드포인트 |
| OB 3.0 JSON-LD 출력 | ❌ 미구현 | credential 엔드포인트 |
| 배지 이미지 업로드 | ❌ 미구현 | MinIO 연동 |
| Recipient CRUD 핸들러 | ❌ 미구현 | DB만 있음 |
| 취소 목록 (Revocation) | ❌ 미구현 | 1EdTech StatusList |
| DID 지원 | ❌ 미구현 | did:web, did:key |

## 구현 우선순위 (앞으로 할 일)

### Phase 1: 핵심 서명/검증
1. **Issuer에 키 쌍 추가** — DB에 공개키/비밀키 컬럼 추가
2. **Ed25519 서명 모듈** — `pkg/openbadge/proof.go`
3. **JSON-LD 정규화** — RDFC-1.0 라이브러리 연동
4. **서명 생성** — assertion 발급 시 자동 서명
5. **서명 검증** — verify 엔드포인트 실제 검증 로직

### Phase 2: OB 3.0 준수
6. **JSON-LD 출력** — 완전한 AchievementCredential 포맷
7. **DID 지원** — did:web 기반 발급자 식별
8. **Recipient 해시** — 이메일 해시로 프라이버시 보호

### Phase 3: 운영
9. **배지 이미지 업로드** — MinIO 연동
10. **취소 목록** — StatusList2021 or 1EdTech 방식
11. **Verifiable Presentation** — 배지 묶어서 제출
