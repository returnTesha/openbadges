# 기술 개요

The Badge 시스템은 **1EdTech Open Badges 3.0(OB 3.0)** 표준을 기반으로 설계된 디지털 배지 발급·검증 플랫폼이다. 여러 대학의 학사 시스템(CPS)과 연동하여, 학습자가 비교과 이수 또는 역량 진단을 완료하면 표준 규격의 디지털 배지를 발급한다.

## 핵심 기술 원칙

| 원칙 | 설명 |
|---|---|
| DID 방식 | `did:web` 기본 채택. 발급기관 소멸 시에도 공개키 조회가 가능하도록 Polygon 스마트컨트랙트에 공개키 앵커링 |
| 블록체인 불필요 | 공개키 암호학(Ed25519)으로 위변조 불가성 달성 |
| 오프라인 검증 가능 | 공개키만 있으면 서버 없이도 배지 진위 확인 가능 |
| 표준 호환성 | W3C Verifiable Credentials v2.0 및 OB 3.0 규격 준수 |
| 서버 기술 | Go(Fiber 프레임워크) 기반 단일 서버 구성 |
| 개인키 관리 | HashiCorp Vault에서 관리하며, 배지 JSON에 Polygon 컨트랙트 주소를 포함하여 발급기관 없이도 자체 검증 가능 |

> **참고:** OB 3.0은 '블록체인에 저장'하는 방식이 아니라, 수학적 서명(Ed25519)으로 위변조를 방지한다. 블록체인과 동일한 '위변조 불가' 특성을 훨씬 낮은 인프라 비용으로 달성할 수 있다.

## OB 3.0과 블록체인의 차이

Open Badges 3.0 도입 초기에 '블록체인에 저장하는 것 아닌가'라는 질문이 자주 발생한다. 두 방식은 **신뢰의 근거가 근본적으로 다르다**.

| 구분 | 블록체인 | Open Badges 3.0 |
|---|---|---|
| **신뢰 근거** | 네트워크 합의 (수천 노드) | 수학적 서명 (키 쌍) |
| **저장소** | 분산 원장 | 어디든 (파일, DB, 이메일, USB) |
| **검증 방법** | 체인 조회 | 공개키로 서명 검증 |
| **비용** | 가스비 등 발생 | 없음 |
| **인프라** | 노드 운영 필요 | 서버 하나로 충분 |
| **공통점** | 위변조 불가 | 위변조 불가 |

- **블록체인**: '수천 명이 같은 장부를 갖고 있어서 고칠 수 없다'는 원리
- **OB 3.0**: '수학적으로 서명했으니 고칠 수 없다'는 원리
- 목표(위변조 불가)는 같지만 방식이 완전히 다르다.

## 암호화(Encryption)와 서명(Signature)의 차이

OB 3.0을 처음 접하면 '배지를 암호화하는 것인가?'라고 오해하기 쉽다. 그러나 Open Badges는 **암호화가 아니라 서명을 사용**한다.

| 구분 | 암호화 (Encryption) | 서명 (Signature) |
|---|---|---|
| **목적** | 데이터를 숨긴다 | 데이터가 진짜임을 증명한다 |
| **효과** | 열쇠 없이는 내용을 읽을 수 없게 만듦 | 누가, 언제 만들었는지를 증명 |

배지의 내용(이름, 발급 기관, 이수 기준 등)은 누구나 읽을 수 있다. 서명은 **'이 배지를 진짜 우리 기관이 발행했다'**는 사실을 수학적으로 증명하는 장치이다.
